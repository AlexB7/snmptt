SNMPTT (SNMP Trap Translator) is an SNMP trap handler written in Perl for use with the Net-SNMP / UCD-SNMP snmptrapd program (www.net-snmp.org).

6/16/2007

SNMPTT v1.2 has been released

Various database improvements have been added such as support for proper date/time data types, statistics logging and custom columns. The formatting for unknown_trap_exec can now be changed and support has been added to MATCH for bitwise AND. A new log file has been added for logging system messages to a text file for users without access to syslog. Various bug fixes have been applied including support for Net-SNMP 5.4 traphandle, daemon mode pid file, REGEX captures, MATCH modifiers and others.

See the Change Log for more details
1/5/2007

SNMPTT v1.2beta2 has been released

A second beta for SNMPTT v1.2 has been released. A list of new features is available on the SourceForge page.
1/17/2006

SNMPTT v1.1 has been released

A PREEXEC snmptt.conf file option has been added to allow an external program to be run before processing the FORMAT and EXEC lines. The NODES statement now allows for both positive and negative matches and the MATCH statement will now accept any variable name instead of only enterprise variables. A default EXEC command can be set for unknown traps and various bugs have been fixed.

See the Change Log for more details
5/1/2005

Sys Admin Magazine article

The March edition of Sys Admin Magazine features Francois Meehan's article on SNMP Trap Handling with Nagios. The article explains an alternative way of integrating SNMPTT and Nagios using SEC. The article can now be viewed online.
9/1/2004

SNMPTT v1.0 has been released

Various improvments were added for SQL database connections including database auto-reconnect and logging of unknown traps. New variable substitutions were added for the trap description and formatting such as newline, form feed etc. Statistical info on the number of traps received/processed is now logged plus various bug fixes including symbolic traps issues, MIB conversion and FORMAT/EXEC line parsing.

See the Change Log for more details

Net-SNMP 5.1.2 released

4/5/2004

Net-SNMP 5.1.1 released

Net-SNMP 5.1.1 has been released and is now the recommended version to use with SNMPTT. V5.1.1 contains all the recommended Perl patches and the problems reported in 5.1 have been fixed. Net-SNMP source and binaries are available from the Net-SNMP download page.
12/5/2003

Net-SNMP 5.1 problems

There seems to be a problem with the trap handler in Net-SNMP 5.1. Until it is fixed, please use Net-SNMP 5.0.9 or lower with SNMPTT.
11/3/2003

SNMPTT v0.9 has been released

The process name for syslog output has been changed to 'snmptt[PID]' for traps and 'snmptt-sys[PID]' for system messages. The UID used by snmptt can now be changed after startup, and some bugs have been fixed.

See the Change Log for a complete listing of changes.
10/23/2003

Web site changes

This site has been redesigned once again using a combination of the 2 column CSS from The Layout Reservoir and some parts of the Nagios web site CSS.

All pages are now valid XHTML 1.0 Transitional which means the site should work in all browsers including wireless devices and screen readers. Some older browsers may not show the correct layout, but the contents of the site should be accessible.
10/16/2003

Web site launch

Although long overdue, the domain snmptt has been registered, and this new web site has been created. The layout and look of the site is based on the Nagios web site, so special thanks to Ethan!

The old web page was the readme.html file included in the package, and is available from the Documentation menu item on the left.

Over time the documentation will be cleaned up with hopefully a multi-page format and a tutorial section to help new users as the size of current documentation can be a bit intimidating at first. The basic use of SNMPTT is actually quite simple, but things can get complex when using new features such as REGEX.
09/16/2003

SNMPTT v0.8 has been released

New features have been added such as the MATCH keyword to match events based on enterprise variables, the REGEX keyword now supports captures and the i, g, and e modifiers for complex regular expressions, improved PostgreSQL support, DNS lookups, and remote MySQL and PostgreSQL support. Some variable substitutions have been added or changed, and Nagios/ Netsaint integration documentation has been added along with various bug fixes.

See the Change Log for a complete listing of changes.
